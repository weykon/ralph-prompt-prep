Why Ralph Prompt Prep?

Ralph Wiggum is a powerful iterative development technique where Claude repeatedly works on the same prompt, seeing its previous work through files and git history. But writing effective Ralph prompts is challenging.

❌ Common Problems

  • Vague goals ("make it better")
  • Scope too broad (trying to do everything)
  • No validation strategy
  • Missing completion criteria
  • Unclear success metrics

✅ Ralph Prompt Prep Solution

  • Clear, testable goals
  • Focused scope (In/Out defined)
  • Concrete validation steps
  • Completion promise markers
  • Quantifiable success criteria

The skill will guide you through an interactive conversation to build your prompt.

Answer the questions and get a structured prompt automatically generated.

Method 3: Manual Template

Copy the template from quick_reference.md and fill in each section.

📖 What Makes a Good Ralph Prompt?

  • Clear Goal: One sentence that describes what you want to accomplish
  • Testable Success Criteria: Not "works well", but "tests pass" or "success rate >85%"
  • Defined Scope: Explicit In Scope and Out of Scope lists
  • Validation Steps: Specific commands to run and things to check
  • Completion Promise: The <promise> tag that signals done
  • Reasonable Iterations: 5-20 iterations for most tasks
  • Priority Levels: P0 (must), P1 (important), P2 (optional)

💡 Example: Before & After

Before (Vague)

"I want to improve the remote session system and make it better with good UX and maybe add some features for collaboration..."

After (Structured)

Goal: Implement remote session management with user visibility

Success Criteria:

  • Users can see who's controlling their session
  • Accept/reject control requests work
  • All integration tests pass

Validation: Run cargo test, check logs, manual test flow

Completion: <promise>REMOTE_SESSION_COMPLETE</promise>
